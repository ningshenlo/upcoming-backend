Upcoming Games 项目核心目标说明
1. 项目一句话定位
Upcoming Games 是一个面向全球玩家的游戏发售日历与新游戏机会雷达。
它表面上是一个游戏发售日历网站，帮助用户查询即将发布的游戏、发售日期、平台、demo、beta、预载、
 launch time 和订阅服务上架信息。
它底层上是一个游戏内容情报系统，用来发现新词流量、Reddit 社区痛点、程序化 SEO 机会和 micro-tool 启动机
会。
一句话：
Upcoming Games = Game Release Calendar + Launch Time Tracker + New Game 
Opportunity Radar
2. 项目的核心判断

11. 关键设计原则
11.1 数据优先，而不是页面优先
先建立游戏实体、事件、平台、来源，再生成页面。
页面只是数据的不同视图。
11.2 事件优先，而不是单一发售日期
游戏生命周期中有多个重要事件：
公布
预告片
 demo 
beta
预载
early access
正式发售
DLC
延期
加入订阅服务
系统必须能追踪这些事件。
15
11.3 证据优先
每个日期、事件、趋势和机会都应该有来源。
没有来源的数据可以存在，但必须降低 confidence。
11.4 不生成无价值页面
程序化 SEO 的风险是页面太薄。
系统应该根据数据完整度和搜索意图决定页面是否 index。
11.5 前台和后台分离
前台服务用户和搜索引擎。
后台服务机会发现和研究决策。
11.6 趋势信号驱动行动
最终目标不是展示数据，而是告诉我们：
该生成什么页面？
该研究哪个 Reddit 帖？
该做哪个小工具？
该关注哪个游戏？

13. 项目核心结论
Upcoming Games 是一个非常适合程序化 SEO 的项目，因为游戏领域具有天然结构化属性：
游戏实体
平台
时间
发行事件
搜索意图
社区讨论
工具需求
它的核心机会在于：
用游戏日历承接稳定搜索流量；
用 launch time / demo / beta / preload 页面承接长尾流量；
用 Reddit 和趋势信号发现新游戏痛点；
用 micro-tool 抢占社区启动和品牌词；
用结构化数据长期积累壁垒。
最终项目不是一个单纯的   upcoming games 关键词站，而是一个游戏领域的机会发现系统。
