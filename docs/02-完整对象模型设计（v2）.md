## 1. 对象总览

系统共设计 **15 个核心对象**：

| # | 对象名 | 中文名 | 阶段 | 说明 |
|---|-------|-------|------|------|
| 1 | `Game` | 游戏实体 | MVP | 系统核心，代表一款游戏 |
| 2 | `ReleaseEvent` | 发行事件 | MVP | 游戏的所有时间事件 |
| 3 | `Platform` | 平台 | MVP | 游戏平台/商店/订阅服务 |
| 4 | `Company` | 公司实体 | MVP | 发行商、开发商、工作室等公司 |
| 5 | `Source` | 数据源 | MVP | 信息来源与可信度管理 |
| 6 | `Signal` | 趋势信号 | MVP | 外部热度信号 |
| 7 | `Page` | SEO页面 | MVP | 程序化页面索引与控制 |
| 8 | `DataJob` | 数据任务 | MVP | 采集任务状态追踪 |
| 9 | `Alert` | 数据告警 | MVP | 数据冲突/异常告警 |
| 10 | `Keyword` | 关键词 | Phase 2 | 关键词对象与评分 |
| 11 | `Opportunity` | 机会 | Phase 2 | 行动机会发现与管理 |
| 12 | `Community` | 社区 | Phase 2 | Reddit等社区对象 |
| 13 | `StoreLink` | 商店链接 | Phase 2 | 各平台商店页 |
| 14 | `Franchise` | 游戏系列 | Phase 2 | 游戏IP/系列 |
| 15 | `SearchTrend` | 搜索趋势 | Phase 2 | Google Trends时序数据 |

## 2. 对象关系图

```
Game
├── has many ReleaseEvents        (事件时间线)
├── has many StoreLinks           (商店页)
├── has many Signals              (趋势信号)
├── has many Keywords             (关键词)
├── has many Pages                (SEO页面)
├── has many Opportunities        (行动机会)
├── belongs to many Platforms     (平台)
├── belongs to many Companies     (发行商/开发商角色)
├── belongs to many Genres/Tags   (类型标签)
├── belongs to Franchise          (所属系列)
└── relates to many Communities   (相关社区)

Company
├── may publish many Games
└── may develop many Games

ReleaseEvent
├── belongs to Game
├── relates to Platforms
├── backed by Sources
└── may trigger Alerts (日期冲突)

Signal
├── belongs to Game
├── backed by Source
└── feeds into Opportunity score

Keyword
├── optionally belongs to Game
├── may generate Page
└── may generate Opportunity

Page
├── optionally belongs to Game
├── optionally belongs to Platform
└── targets Keywords

Opportunity
├── optionally belongs to Game
├── optionally belongs to Keyword
└── uses Signals/Sources/Communities as evidence

DataJob
├── belongs to Source
└── may create Alerts

SearchTrend
├── belongs to Game (or free-floating keyword)
└── feeds into Signal
```

## 3. Game — 游戏实体对象

### 3.1 TypeScript 类型定义

```typescript
type Game = {
  id: string;
  slug: string;                    // URL 友好的唯一标识，如 "gta-6"
  title: string;                   // 显示标题
  canonicalTitle: string;          // 官方标准标题（用于去重）
  aliases: string[];               // 别名数组，如 ["GTA VI", "Grand Theft Auto 6"]

  // 内容字段
  description?: string;            // 详细描述（来自 IGDB 或人工编辑）
  shortDescription?: string;       // 简短摘要（用于卡片展示，≤160字符）
  coverImageUrl?: string;          // 封面图 URL（IGDB cover）
  headerImageUrl?: string;         // 横幅大图（Steam header）
  screenshotUrls?: string[];       // 截图列表
  trailerUrl?: string;             // 最新预告片 URL（YouTube）
  trailerThumbnailUrl?: string;    // 预告片缩略图

  // 状态
  status:
    | "rumored"                    // 传言中
    | "announced"                  // 已公布
    | "upcoming"                   // 即将发售
    | "early_access"               // 抢先体验中
    | "released"                   // 已发售
    | "delayed"                    // 延期
    | "cancelled";                 // 取消

  // 关联实体 ID
  developerIds: string[];
  publisherIds: string[];
  franchiseId?: string;
  genreIds: string[];
  tagIds: string[];
  platformIds: string[];           // 已确认平台列表

  // 日期概览（详细时间线用 ReleaseEvent 管理）
  firstAnnouncedAt?: string;       // ISO8601
  firstReleaseDate?: string;       // 最早发售日期（任意平台）
  primaryReleaseDate?: string;     // 主力平台发售日（用于日历展示）

  // 外部ID（用于数据关联）
  igdbId?: number;                 // IGDB 数据库 ID
  steamAppId?: number;             // Steam App ID
  metacriticSlug?: string;         // Metacritic 页面标识

  // 热度与评分（由系统计算，定期更新）
  trendScore?: number;             // 综合热度分 0-100（派生字段）
  opportunityScore?: number;       // 综合机会评分 0-100（派生字段）
  searchVolume7d?: number;         // 近7天搜索量（来自 Keyword 汇总）
  redditMentionCount7d?: number;   // 近7天 Reddit 提及次数

  // 数据管理
  dataCompleteness?: number;       // 数据完整度 0-100
  needsReview?: boolean;           // 是否需要人工审核
  lastScrapedAt?: string;          // 最后一次抓取时间

  createdAt: string;
  updatedAt: string;
};
```

### 3.2 设计说明

- `primaryReleaseDate` 仅用于列表展示，不代表所有平台
- 所有实际发行事件（包括延期、demo、beta）存储在 `ReleaseEvent` 中
- `trendScore` 和 `opportunityScore` 是计算字段，由后台 Job 定期刷新
- `igdbId` 和 `steamAppId` 用于增量同步，不可重复

## 4. ReleaseEvent — 发行/内容事件对象

### 4.1 TypeScript 类型定义

```typescript
type ReleaseEvent = {
  id: string;
  gameId: string;

  eventType:
    | "announcement"               // 游戏公布
    | "trailer"                    // 预告片发布
    | "showcase"                   // 直播展示（如 Nintendo Direct）
    | "demo"                       // Demo 版本
    | "beta"                       // Beta 测试（公测/封测）
    | "early_access"               // 抢先体验开放
    | "preload"                    // 预载开始
    | "release"                    // 正式发售
    | "dlc"                        // DLC/扩展包
    | "major_update"               // 重大更新
    | "delay"                      // 延期公告
    | "delisting"                  // 下架
    | "game_pass_addition"         // 加入 Xbox Game Pass
    | "ps_plus_addition"           // 加入 PS Plus
    | "subscription_removal";      // 从订阅服务移除

  title: string;                   // 事件标题，如 "PS5 正式发售"
  description?: string;

  // 时间信息（精度可变）
  date?: string;                   // ISO8601 日期，如 "2026-09-17"
  time?: string;                   // 时间，如 "00:00" (本地时间)
  timezone?: string;               // 时区，如 "America/Los_Angeles"
  launchTimeUTC?: string;          // 标准化 UTC 时间（用于倒计时）
  dateAccuracy:
    | "exact"                      // 精确日期
    | "week"                       // 本周内
    | "month"                      // 本月内
    | "quarter"                    // 本季度
    | "year"                       // 年份已知
    | "unknown";                   // 未知

  // 平台和地区
  platformIds: string[];
  region?:
    | "global"
    | "na"                         // 北美
    | "eu"                         // 欧洲
    | "jp"                         // 日本
    | "asia"
    | "other";

  // 置信度
  status:
    | "rumored"                    // 传言（confidence < 30）
    | "leaked"                     // 泄露（confidence 30-60）
    | "announced"                  // 官方宣布（confidence 60-85）
    | "confirmed"                  // 官方确认+多源验证（confidence > 85）
    | "changed"                    // 已变更
    | "cancelled"                  // 已取消
    | "released";                  // 已发生

  confidence: number;              // 0-100，综合置信度

  // 来源追踪
  sourceIds: string[];
  sourceUrls: string[];

  // 变更历史
  previousDate?: string;           // 变更前日期（延期时记录）
  changedAt?: string;              // 变更时间

  // 冲突检测
  conflictsWith?: string[];        // 与哪些 Source ID 存在冲突

  createdAt: string;
  updatedAt: string;
  lastVerifiedAt?: string;         // 最后人工/自动验证时间
};
```

### 4.2 设计说明

**时区处理原则：**
- 对外展示时，根据用户浏览器时区转换
- `launchTimeUTC` 是倒计时的基准时间
- 如果只知道日期不知道具体时间，`launchTimeUTC` 留空

**置信度计算规则：**
```
confidence = 基础分 × 来源质量加权

基础分：
- official_site/store = 90
- publisher/platform_news = 80
- press = 65
- reddit = 40
- rumored = 20

多来源验证加成：
- 2个来源互相印证 +10
- 3个及以上 +15
```

### 8.2 页面质量评分规则

```typescript
function calculateContentQualityScore(page: Page): number {
  let score = 0;

  if (page.hasReleaseDate)      score += 20;
  if (page.hasLaunchTime)       score += 15;
  if (page.hasDescription)      score += 15;
  if (page.hasCoverImage)       score += 10;
  if (page.hasScreenshots)      score += 10;
  if (page.hasTrailer)          score += 15;
  if (page.eventCount >= 3)     score += 10;
  if (page.eventCount >= 5)     score += 5;

  return score; // 最高 100
}

// noindex 规则：
// score < 30 → noindex
// score >= 30 AND hasReleaseDate → indexable
```
